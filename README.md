# .github — Sand Point Studios default community health files

This repository contains the default templates and policy files inherited
by every public/private repo owned by `teaganwins-dev`. Edit a file here
once and it applies portfolio-wide (unless an individual repo provides
its own override).

What lives here:
- **CONTRIBUTING.md** — canonical Git workflow conventions for SPS projects
- (more to come: SECURITY.md, ISSUE_TEMPLATE/, PULL_REQUEST_TEMPLATE.md)

How GitHub uses this:
- A repo without its own `CONTRIBUTING.md` shows this one when contributors
  click "Contributing" or open a PR.
- A repo *with* its own version overrides this one (no fallback merge).
  So: prefer to put canonical content here and remove per-repo copies
  unless a repo genuinely needs a project-specific version.

When a contributor reads or runs into a workflow question, this is the
source of truth.
